# Screenshots

Auto-generated by `python3 scripts/capture_screenshots.py` after `python3 -m linkedin_mcp.web` is running on `127.0.0.1:8000`.

The script uses [Playwright](https://playwright.dev/python/) to load each of the 11 Tailwind+Alpine panels in both light and dark themes and writes full-page PNGs into `docs/screenshots/`.

## How to capture

### 1. Install Playwright

```bash
pip install playwright
python -m playwright install chromium
```

### 2. Start the web server (one terminal)

```bash
python3 -m linkedin_mcp.web
# server is up on http://127.0.0.1:8000
```

### 3. Run the capture script (another terminal)

```bash
chmod +x scripts/capture_screenshots.py   # one-time
python3 scripts/capture_screenshots.py
# → Done: 22 screenshots saved to docs/screenshots/
```

### 4. (Optional) Re-generate on UI changes

Re-run step 3 after modifying any panel under `static/{name}.html` or its Alpine/Tailwind wiring.

## Embedding in docs / GitHub

Markdown:

```markdown
![Dashboard (light)](docs/screenshots/dashboard-light.png)
![Dashboard (dark)](docs/screenshots/dashboard-dark.png)
```

GitHub will render both PNGs in light and dark mode viewers.

## Tips

- The script sets `localStorage.theme` and toggles the `dark` class on `<html>` to bypass the OS preference — useful for deterministic captures in CI.
- For higher-DPI captures, edit `scripts/capture_screenshots.py` and add `device_scale_factor=2` to `new_context()`.
- If a panel needs auth, log in once in a Chromium profile and pass `storage_state="auth.json"` to `new_context()`.
